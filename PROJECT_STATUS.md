# Narchives - Project Status

**Last updated:** 2026-03-23

## Overview

Android app (Kotlin/Jetpack Compose) for browsing web page archives published via Nostr (kind 30041 events) and stored on Blossom media servers. Functions like an RSS reader for WACZ web archives.

**Spec file:** `nostr-archive-reader-spec.md`

---

## Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Nostr library | **Amethyst Quartz** (`com.vitorpamplona.quartz:quartz-android:1.06.0`) | Pure Kotlin, battle-tested (powers Amethyst), includes relay client, bech32, sig verification, Blossom event types. No need to roll our own. |
| WebSocket impl | **BasicOkHttpWebSocket** (bundled in Quartz's jvmAndroid source set) | Already in the library. No custom adapter needed. |
| Dev environment | **Nix flake + direnv** | JDK 17, Android SDK (34/35/36), Gradle 8.11.1, nak 0.17.4. Fully reproducible. |
| DI | **Manual (AppContainer)** | Simple, no Hilt/Dagger. |
| Package name | `com.narchives.reader` | App name: Narchives |

## CURRENT BLOCKER: Gradle Build Fix

All code is written. The build is failing on a **Kotlin version compatibility issue** in `app/build.gradle.kts`. Here's the specific problem and fix needed:

### Problem
Quartz 1.06.0 was compiled with **Kotlin 2.3.20**, which requires:
- **compileSdk 36** (done - already updated)
- **AGP 8.9.3** (done - already updated)
- **Kotlin 2.3.20** (done - already updated in root `build.gradle.kts`)
- **KSP 2.3.6** (done - already updated)

The remaining error is in `app/build.gradle.kts` line 43:
```
kotlinOptions {
    jvmTarget = "17"    // <-- ERROR: deprecated in Kotlin 2.3.x
}
```

### Fix needed
Replace the `kotlinOptions` block with the new `compilerOptions` DSL:
```kotlin
// Remove this:
kotlinOptions {
    jvmTarget = "17"
}

// Replace with:
compilerOptions {
    jvmTarget.set(org.jetbrains.kotlin.gradle.dsl.JvmTarget.JVM_17)
}
```

After this fix, there will likely be Kotlin source-level compile errors to fix (import paths for Quartz classes, etc.), but the Gradle configuration itself should pass.
